2.6.2 Remarks

The State<'UserState> class represents the parser state for Parser<'Result,'UserState> functions.

Each State instance consists of two parts:

  • the input stream position, represented in the form of a CharStream.Iterator instance, a line and column count and a stream name, and
  • a value of the freely definable 'UserState type.

The State type is intended to be used as an immutable type. Its methods do not mutate existing State instances, instead they return new State instances with changed data. The State type implements structural equality.

To advance the State position you can either use one of the Advance(To) methods or one of the Skip… methods. The former methods require you to manually register any skipped newlines by specifying appropriate extra arguments, while the latter methods automatically detect standard newlines ("\n", "\r\n" or "\r").

⇧ 
val Iter: CharStream.Iterator

The CharStream iterator pointing to the current char in the input stream.

Note

Even though this member is actually declared as a mutable field (due to optimization reasons), you should never mutate it. Hence, you should not assign a new value to the Iter member of an existing State instance or directly call one of the _Increment or _Decrement methods directly on the Iter member of an existing State instance.

⇧

⇧

The current line number.

⇧

The stream index of the first char of the current line, i.e. Index ‐ Column + 1.

⇧

The current column number, or, to be technically precise, the 1‐based index of the current UTF‐16 char in the line, i.e. Index ‐ LineBegin + 1.

⇧ 
member Position: Position

Is equivalent to new Position(StreamName, Index, Line, Column).

⇧ 
member UserState: 'u

The current user state value.

Note

FParsec relies on the user state type 'u being immutable (or at least being treated as an immutable type). If you want to change the user state value, you should create a new State instance with a new user state value using WithUserState or one of the Advance(To) methods. If you directly mutate the user state value or a nested member, you break FParsec’s ability do undo the state change during backtracking.

⇧ 
member ReadUntil: State<'u> -> string

Returns a string with all the chars between the position of this state (inclusive) and the position of the state argument (exclusive). If the state argument does not indicate a position after the position of this state, the returned string is empty.

In the returned string any newline ("\n", "\r\n" or "\r") is normalized to "\n", assuming that the line count in the state argument properly accounts for all newlines between the two states.

An ArgumentOutOfRangeException is thrown if the states are associated with different CharStream instances.

⇧ 
member SkipWhitespace: unit -> State<'u>

Skips over a space (' '), a tab ('\t') or a newline ("\n", "\r\n" or "\r"). Returns a new State positioned after the whitespace, or the old State (this) if there is no whitespace.

⇧ 
member SkipNewline: unit -> State<'u>

Skips over a newline ("\n", "\r\n" or "\r"). Returns a new State positioned after the newline, or the old State (this) if there is no newline.

⇧ 
member SkipRestOfLine: skipNewline: bool -> State<'u>

Skips over any chars before the next newline ("\n", "\r\n" or "\r") or the end of the stream. If the given bool is true and a newline is present, the newline is also skipped. Returns a new State positioned after the skipped chars, or the old State (this) if no chars were skipped.

⇧ 
member SkipRestOfLine: skipNewline: bool * string out -> State<'u>

Behaves like SkipRestOfLine(skipNewline), except that a string with the skipped chars is assigned to the additional output parameter (without a newline).

⇧ 
member SkipCharOrNewline: unit -> State<'u>

Skips over a newline ("\n", "\r\n" or "\r") or any other single char. Returns a new State positioned after the skipped char(s), or the old State (this) if the stream has already reached the end.

⇧ 
member SkipCharsOrNewlines: nMax: int -> State<'u>

Skips over up to nMax chars. Returns a new State positioned after the skipped chars, or the old State (this) if the stream has already reached the end.

Note that the number of actually skipped chars can be less than nMax if the end of the stream is reached after less than nMax chars.

This method treats all newlines ("\n", "\r\n" or "\r") as a single char.

An ArgumentOutOfRangeException is thrown if nMax is negative.

⇧ 
member SkipCharsOrNewlines: nMax: int * string out -> State<'u>

Behaves like SkipCharsOrNewlines(nMax), except that a string with the skipped chars is assigned to the additional output parameter (with all newlines normalized to '\n').

⇧ 
member SkipCharsOrNewlines: nMax: int * int out -> State<'u>

Behaves like SkipCharsOrNewlines(nMax), except that the number of skipped chars is assigned to the additional output parameter (with all newlines counted as a single char).

⇧ 
member SkipCharsOrNewlinesWhile:
    f: (char -> bool) -> State<'u>

Skips over a sequence of chars that satisfy the predicate function f. Stops at the first char for which f returns false. Returns a new State positioned after the skipped chars.

If f returns false for the first char or if the stream has already reached the end, the old State (this) is returned.

This method treats all newlines ("\n", "\r\n" or "\r") in the input as a single char '\n'.

Caution

The function predicate f must not access the underlying CharStream (i.e. State.Stream) itself, because SkipCharsOrNewlinesWhile relies on f not having any side‐effect on the internal state of the stream.

⇧ 
member SkipCharsOrNewlinesWhile:
    f: (char -> bool) * string out -> State<'u>

Behaves like SkipCharsOrNewlinesWhile(f), except that a string with the skipped chars is assigned to the additional output parameter (with all newlines normalized to '\n').

⇧ 
member SkipCharsOrNewlinesWhile:
    f: (char -> bool) * nMin: int * nMax: int -> State<'u>

Skips over a sequence of up to nMax chars that satisfy the predicate function f. If in total not less than nMin chars were skipped, a new State positioned after the skipped chars is returned. Otherwise the old State (this) is returned.

This method treats all newlines ("\n", "\r\n" or "\r") in the input as a single char '\n'.

An ArgumentOutOfRangeException is thrown if nMax is negative.

Caution

The function predicate f must not access the underlying CharStream (i.e. State.Stream) itself, because SkipCharsOrNewlinesWhile relies on f not having any side‐effect on the internal state of the stream.

⇧ 
member SkipCharsOrNewlinesWhile:
    f: (char -> bool) * nMin: int * nMax: int * string out -> State<'u>

Behaves like SkipCharsOrNewlinesWhile(f, nMin, nMax), except that a string with the skipped chars is assigned to the additional output parameter (with all newlines normalized to '\n').

⇧ 
member SkipCharsOrNewlinesWhile:
    f1: (char -> bool) * f: (char -> bool) -> State<'u>

Behaves like SkipCharsOrNewlinesWhile(f), except that the first char of the skipped string must satisfy f1 instead of f.

⇧

    f1: (char -> bool) * f: (char -> bool) * string out -> State<'u>

Behaves like SkipCharsOrNewlinesWhile(f1, f), except that a string with the skipped chars is assigned to the additional output parameter (with all newlines normalized to '\n').

⇧ 
member SkipCharsOrNewlinesWhile:
    f1: (char -> bool) * f: (char -> bool) * nMin: int * nMax: int -> State<'u>

Behaves like SkipCharsOrNewlinesWhile(f, nMin, nMax), except that the first char of the skipped string must satisfy f1 instead of f.

⇧ 
member SkipCharsOrNewlinesWhile:
    f1: (char -> bool) * f: (char -> bool) * nMin: int * nMax: int
  * string out -> State<'u>

Behaves like SkipCharsOrNewlinesWhile(f1, f, nMin, nMax), except that a string with the skipped chars is assigned to the additional output parameter (with all newlines normalized to '\n').

⇧ 
member SkipToString: str: string * nMax: int * bool out -> State<'u>

Skips over all the chars before the first occurrence of the string argument or the end of the stream, but not over more than nMax chars. Assigns true to the output parameter if the string was found, otherwise false. Returns a new State positioned after the skipped chars, or the old State (this) if no chars were skipped.

This method treats all newlines ("\n", "\r\n" or "\r") in the input as a single char. However, no newline normalization takes place when the string is matched. Hence the string argument should either contain no newlines or only in the form they occur in the stream.

This method throws

  • an ArgumentException, if the string argument is empty, and
  • an ArgumentOutRangeException, if nMax is negative.
⇧ 
member SkipToString: str: string * nMax: int * string out -> State<'u>

Behaves like SkipToString(str, nMax, outBool), except that its output parameter is a string instead of a boolean. If str is found, a string with the skipped chars is assigned to this output parameter (with all newlines normalized to '\n'), otherwise null is assigned to the output parameter.

⇧ 
member SkipToStringCI:
    caseFoldedString: string * nMax: int * bool out -> State<'u>

Skips over all the chars before the first case‐insensitive occurrence of the string argument or the end of the stream, but not over more than nMax chars. Assigns true to the output parameter if the string was found, otherwise false. Returns a new State positioned after the skipped chars, or the old State (this) if no chars were skipped.

Note

While the chars in the CharStream are case‐folded before they are matched, the chars in the string argument are assumed to already be case‐folded (e.g. with the help of FoldCase). Please see the CharStream documentation for more details on the notion of case‐folding and case‐insensitive matching applied by this method.

This method treats all newlines ("\n", "\r\n" or "\r") in the input as a single char. However, no newline normalization takes place when the string is matched. Hence the string argument should either contain no newlines or only in the form they occur in the stream.

This method throws

  • an ArgumentException, if the string argument is empty, and
  • an ArgumentOutRangeException, if nMax is negative.
⇧ 
member SkipToStringCI:
    caseFoldedString: string * nMax: int * string out -> State<'u>

Behaves like SkipToStringCI(caseFoldedString, nMax, outBool), except that its output parameter is a string instead of a boolean. If str is found, a string with the skipped chars is assigned to this output parameter (with all newlines normalized to '\n'), otherwise null is assigned to the output parameter.
